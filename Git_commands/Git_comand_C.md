## 2.git cherry-pick เมื่อพิจารณาคอมมิตที่มีอยู่ตั้งแต่หนึ่งคอมมิตขึ้นไป ให้ใช้การเปลี่ยนแปลงที่แต่ละคอมมิตแนะนำ และบันทึกคอมมิตใหม่สำหรับแต่ละคอมมิต สิ่งนี้ต้องการให้แผนผังการทำงานของคุณสะอาด
![cherry-pick](https://github.com/VisawaPRO/Git_A-Z_Mission_65030229/assets/144195555/3b875ae1-95d3-4194-bf90-bd51780f2ebe)

## 3.git citool อินเทอร์เฟซแบบกราฟิกที่ใช้ Tcl/Tk เพื่อตรวจสอบไฟล์ที่แก้ไข จัดทำดัชนี ป้อนข้อความคอมมิต และบันทึกคอมมิตใหม่ลงในสาขาปัจจุบัน อินเทอร์เฟซนี้เป็นทางเลือกแทนโปรแกรมคอมไพล์ที่มีการโต้ตอบน้อย

## 4.git clean อินเทอร์เฟซแบบกราฟิกที่ใช้ Tcl/Tk เพื่อตรวจสอบไฟล์ที่แก้ไข จัดทำดัชนี ป้อนข้อความคอมมิต และบันทึกคอมมิตใหม่ลงในสาขาปัจจุบัน อินเทอร์เฟซนี้เป็นทางเลือกแทนโปรแกรมคอมไพล์ที่มีการโต้ตอบน้อย
![image](https://github.com/VisawaPRO/Git_A-Z_Mission_65030229/assets/144195555/d8a74eb8-150e-49bc-9b6f-fe2586d42546)

## 5.git clone โคลนพื้นที่เก็บข้อมูลลงในไดเร็กทอรีที่สร้างขึ้นใหม่ สร้างสาขาการติดตามระยะไกลสำหรับแต่ละสาขาในพื้นที่เก็บข้อมูลที่โคลน (มองเห็นได้โดยใช้สาขา git --remotes) และสร้างและตรวจสอบสาขาเริ่มต้นที่แยกจากสาขาที่ใช้งานอยู่ในปัจจุบันของพื้นที่เก็บข้อมูลโคลน

## 6.git commit สร้างการคอมมิตใหม่ที่มีเนื้อหาปัจจุบันของดัชนีและข้อความบันทึกที่กำหนดซึ่งอธิบายการเปลี่ยนแปลง คอมมิตใหม่นั้นเป็นลูกโดยตรงของ HEAD ซึ่งโดยปกติจะเป็นส่วนปลายของสาขาปัจจุบัน และสาขาจะได้รับการอัปเดตให้ชี้ไปที่สาขานั้น (เว้นแต่ว่าไม่มีสาขาใดที่เกี่ยวข้องกับแผนผังการทำงาน ซึ่งในกรณีนี้ HEAD จะถูก "แยกออก"
![image](https://github.com/VisawaPRO/Git_A-Z_Mission_65030229/assets/144195555/96a9eae5-fd9e-4809-a0d8-1170e9aa3617)

## 7.git config คุณสามารถสืบค้น/ตั้งค่า/แทนที่/ยกเลิกการตั้งค่าตัวเลือกด้วยคำสั่งนี้ จริงๆ แล้วชื่อคือส่วนและคีย์ที่คั่นด้วยจุด และค่าจะถูกหลีกเลี่ยง
![image](https://github.com/VisawaPRO/Git_A-Z_Mission_65030229/assets/144195555/dbb38f1b-be1b-4ddf-baa4-f291aa4093a7)

## 8.git count-objects ซึ่งจะนับจำนวนไฟล์อ็อบเจ็กต์ที่คลายแพ็กและพื้นที่ดิสก์ที่ใช้ เพื่อช่วยคุณตัดสินใจว่าเมื่อใดจึงเป็นเวลาที่เหมาะสมในการแพ็กใหม่

## 9.git cvsexportcommit ส่งออกคอมมิตจาก Git ไปยังการชำระเงิน CVS ทำให้ง่ายต่อการรวมแพตช์จากที่เก็บ Git ไปยังที่เก็บ CVS

## 10.git cvsimport git cvsimport ใช้ cvsps เวอร์ชัน 2 ซึ่งถือว่าเลิกใช้แล้ว มันใช้ไม่ได้กับ cvsps เวอร์ชัน 3 และใหม่กว่า หากคุณกำลังดำเนินการนำเข้าที่เก็บ CVS แบบครั้งเดียว ให้พิจารณาใช้ cvs2git หรือ cvs-fast-export นำเข้าที่เก็บ CVS ไปยัง Git มันจะสร้างพื้นที่เก็บข้อมูลใหม่หรือนำเข้าเพิ่มเติมไปยังพื้นที่เก็บข้อมูลที่มีอยู่

## 11.git cvsserver แอปพลิเคชันนี้เป็นเลเยอร์การจำลอง CVS สำหรับ Git มันมีฟังก์ชั่นการใช้งานสูง อย่างไรก็ตาม ไม่ได้มีการนำวิธีการทั้งหมดไปใช้ และสำหรับวิธีการเหล่านั้นที่นำไปใช้นั้น สวิตช์บางตัวก็ไม่ได้ถูกนำมาใช้

## 12.git cat-file ในรูปแบบแรก คำสั่งจัดเตรียมเนื้อหาหรือประเภทของอ็อบเจ็กต์ในที่เก็บ จำเป็นต้องมีประเภท เว้นแต่ว่าจะใช้ -t หรือ -p เพื่อค้นหาประเภทวัตถุ หรือใช้ -s เพื่อค้นหาขนาดวัตถุ หรือใช้ --textconv หรือ --filters (ซึ่งหมายถึงประเภท "blob")
